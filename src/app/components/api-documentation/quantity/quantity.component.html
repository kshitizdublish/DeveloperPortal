Dana Aftermarket Part Quantity API

Introduction

The Part Quantity API provides detailed quantity information via a simple HTTP REST style API. This allows for faster integration into your existing applications with the programming language of your choice.

The API goes beyond providing the quantity date and quantity and provides detailed manufacturing plant level information for deeper insight and potential logistics optimization.

The API call is secured by the API Key provided to you. This API key is applicable only to you and should be stored securely.

The data format for the response message is provided in either JSON or XML.

See below for the detailed technical specification of the API.

HTTP Request Headers:

Header Field Value Comments
apiKey <API_Key> API Key provided in this developer portal should be specified in place of
<API_Key>.

HTTP Request Parameters

Request Parameter Required Default
Value		 Allowed
Values		 Description
partNumber Yes N/A Part Number can be Dana part number or customer part number. When partNumber is customer specific, isCustomerPartNumber parameter should be specified as ‘YES’. If the partNumber contain non alpha-numeric characters, the parameter value should be URL encoded.
format No format json/xml format
isCustomerPartNumber No NO YES or NO YES – Implies that the partNumber is a customer Interchange part number.
NO – Implies that the partNumberis Dana part number.
orderType No SO SO or EO SO – Stock Order (Parts available on stock)
EO – Emergency Order (Parts available on emergency basis. Enhanced information is available with detailed manufacturing plant level visibility)
qtyRequested No 1 Quantity of the parts required.
